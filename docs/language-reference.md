# SlavLang: полный справочник языка

SlavLang — древнерусскій системный языкъ, держимый какъ самостоятельная
альтернатива C++. Его синтаксис строится не как перевод английских ключевых
слов, а вокруг образов древней Руси:

- зависимости **призываются**;
- исполняемой частью программы правит **Кнѧзь**;
- стандартные возможности разделены между именованными модулями;
- служебные запятые, скобки и символьные операторы не используются.

Документ описывает весь синтаксис, который поддерживает текущий frontend.
Исходные файлы имеют расширение `.slav`, кодировку UTF-8 и обрабатываются
построчно. Регистр ключевых слов не имеет значения.

## Полная минимальная программа

```slav
възвати Ярило

Кнѧзь
    речи "Здравъ, мир!"
коньць
```

`Кнѧзь` — обязательная и единственная точка входа. Все исполняемые команды
находятся между `Кнѧзь` и соответствующим ему словом `коньць`.

Отступы необязательны, но рекомендуются. Точка с запятой не ставится.

## Запрет служебной пунктуации

В коде SlavLang вне строк запрещены:

```text
(  )  [  ]  {  }  ,  =  !  <  >  +  -  *  /  %
```

Вместо них используются слова:

| Классическая запись | SlavLang |
| --- | --- |
| `1, 2, 3` | `1 и 2 и 3` |
| `[1, 2, 3]` | `рядъ 1 и 2 и 3` |
| `длъгота(числа)` | `длъгота числа` |
| `съчѧти(" ", числа)` | `съчѧти " " и числа` |
| `числа[i]` | `числа по i` |
| `значение = 5` | `значение єсть 5` либо `значение се 5` |
| `a == b` | `a воистину b` |
| `a != b` | `a не єсть b` либо `a се не b` |

Запятая внутри строкового значения является обычным текстом и разрешена:

```slav
речи "Здравъ, мир!"
```

Группирующих скобок нет. Для изменения порядка вычислений результат выражения
следует сохранить в промежуточную переменную.

## Призыв модулей

Модуль призывается до `Кнѧзя`:

```slav
възвати Сварога
възвати Ярило
```

Это аналог подключения зависимости, но часть собственной модели SlavLang, а не
калька с `include`, `using` либо `import`.

Текущая стандартная дружина:

| Призыв | Возможности |
| --- | --- |
| `възвати Ярило` | `речи`, `плачь` |
| `възвати Сварога` | `рядъ`, `длъгота`, `съчѧти`, доступ через `по` |

Компилятор принимает имя модуля как идентификатор. Разрешение внешних модулей
и пользовательских дружин ещё не реализовано. Стандартные конструкции проверяют
свой призыв: например, `речи` без `възвати Ярило` является ошибкой.

## Точка входа `Кнѧзь`

В одном файле допускается один `Кнѧзь`:

```slav
възвати Ярило

Кнѧзь
    речи "Начало"
коньць
```

Команды до `Кнѧзя` либо после закрывающего его слова `коньць` запрещены.
При обычном завершении `Кнѧзь` возвращает процессу код `0`.

## Комментарии

Строка, первый непробельный символ которой — `#`, является комментарием:

```slav
# Это комментарий
речи "Эта строка выполняется"
```

Комментарий занимает всю строку. Комментарии после команды дондеже не
поддерживаются.

## Значения

Поддерживаются:

- целые числа: `42`, `минусъ 7`;
- вещественные числа: `3.14`;
- строки: `"текст"`;
- логические значения: `истинно`, `лъжа`;
- ряды: `рядъ 5 и 2 и 9 и 1`.

Строковые литералы поддерживают escape-последовательности `\"`, `\\`, `\n`,
`\r`, `\t`.

## Переменные

Переменная объявляется словом `да`; её тип выводится из значения:

```slav
да счьтъ єсть 0
да имя се "Анна"
да готово єсть истинно
да числа се рядъ 4 и 2 и 8
```

Повторное присваивание:

```slav
счьтъ єсть счьтъ плюсъ 1
имя се "Мария"
```

`єсть` и `се` равнозначны. Они используются за объявления и обычного
присваивания, но не за сравнения.

Тип переменной после объявления не меняется. Имена могут содержать кириллическія и
латинские буквы, цифры и `_`, но не могут начинаться с цифры.

## Ряды

Для работы с рядами призывается Сварог:

```slav
възвати Сварога

Кнѧзь
    да числа єсть рядъ 4 и 2 и 8
коньць
```

Элементы перечисляются словом `и`, без запятых и скобок. Все элементы должны
иметь совместимый тип. Размер ряда после создания не меняется.

Доступ к элементу:

```slav
да первое єсть числа по 0
числа по 1 се 10
```

Индекс начинается с нуля. После `по` указывается число либо имя переменной.
Сложный указъ сначала вычисляется отдельно:

```slav
да слѣдъ єсть указъ плюсъ 1
да значение се числа по слѣдъ
```

Выход за границы ряда завершает программу ошибкой среды выполнения.

## Выражения

Арифметические операции записываются словами:

| Операция | SlavLang |
| --- | --- |
| сложение | `плюсъ` |
| вычитание и отрицание | `минусъ` |
| умножение | `множити` |
| деление | `раздѣлити` |
| останъкъ отъ деления | `останъкъ` |

Сравнения:

| Смысл | Основная форма | Допустимая форма |
| --- | --- | --- |
| равно | `воистину` | — |
| не равно | `не єсть` | `се не` |
| паче | `паче` | — |
| менши | `менши` | — |
| паче либо равно | `не менши` | — |
| менши либо равно | `не паче` | — |

Логические операции:

```text
не  и  либо
```

Приоритет отъ высокого к низкому:

1. `не` и `минусъ` перед одним значением;
2. `множити`, `раздѣлити`, `останъкъ`;
3. `плюсъ`, `минусъ`;
4. сравнения;
5. `и`;
6. `либо`.

Группирующих скобок нет. Неоднозначное вычисление разбивается на шаги:

```slav
да сумма єсть a плюсъ b
да результат се сумма множити c
```

### Составные присваивания

Основные формы:

```slav
счьтъ плюсъ єсть 5
счьтъ минусъ єсть 2
счьтъ множити єсть 3
счьтъ раздѣлити єсть 2
счьтъ останъкъ єсть 4
```

Разрешены более разговорные варианты:

```slav
счьтъ приложити 5
счьтъ отъяти 2
счьтъ множити на 3
счьтъ раздѣлити на 2
счьтъ останъкъ отъ 4
```

Увеличение и уменьшение на единицу:

```slav
счьтъ плюсъ плюсъ
счьтъ минусъ минусъ
счьтъ възрастити
счьтъ умалити
```

Каждая пара форм равнозначна.

## Вывод

Стандартный поток:

```slav
речи "Здравъ"
речи счьтъ
```

Поток ошибок:

```slav
плачь "Файл не найден"
плачь код
```

Обе команды требуют `възвати Ярило` и добавляют перевод строки.

Для совместимости простой текст можно выводить без кавычек:

```slav
речи Здравъ мир
```

Вычисляемые строки лучше заключать в кавычки либо получать встроенной командой,
чтобы запись оставалась однозначной.

## Встроенные команды Сварога

### `длъгота`

Возвращает мѣра ряда:

```slav
да мѣра єсть длъгота числа
```

В выражении команда записывается без скобок:

```slav
да последний єсть длъгота числа минусъ 1
```

### `съчѧти`

Создаёт строку из элементов ряда:

```slav
речи съчѧти " " и числа
```

Первый аргумент — строка-разделитель, второй — рядъ. Аргументы разделяются
словом `и`.

## Условия

```slav
аще температура менши 0
    речи "Мороз"
инако
    речи "Тепло"
коньць
```

Ветка `инако` необязательна. Блок закрывается словом `коньць`.

## Цикл `дондеже`

```slav
да i єсть 0
дондеже i менши 10
    речи i
    i възрастити
коньць
```

## Цикл `за`

```slav
за i отъ 0 до 10
    речи i
коньць
```

Начальная граница включается, конечная — нет. Пример обходит значения отъ `0`
до `9`. Переменная цикла создаётся автоматически и увеличивается на единицу.

Граница может быть выражением:

```slav
за i отъ 0 до длъгота числа
    речи числа по i
коньць
```

## Управление циклом

Внутри `дондеже` и `за` доступны:

```slav
продолжити
прервати
```

`продолжити` начинает следующую итерацию. `прервати` завершает ближайший цикл.

## Завершение программы

```slav
исходъ 0
```

Аргумент — целочисленное выражение, которое становится кодом завершения
процесса.

## Вложенность

Каждый `Кнѧзь`, `аще`, `дондеже` и `за` требует собственного слова `коньць`:

```slav
възвати Сварога
възвати Ярило

Кнѧзь
    за i отъ 0 до длъгота числа
        аще числа по i паче 0
            речи числа по i
        коньць
    коньць
коньць
```

## Полная грамматика команд

```text
программа       := { призыв } "Кнѧзь" { команда } "коньць"
призыв          := "възвати" имя
комментарий     := "#" текст

команда         := комментарий
                 | "да" имя связка выражение
                 | цель связка выражение
                 | цель составное_присваивание выражение
                 | цель изменение_на_единицу
                 | "речи" значение_вывода
                 | "плачь" значение_вывода
                 | "исходъ" выражение
                 | "аще" выражение { команда } [ "инако" { команда } ] "коньць"
                 | "дондеже" выражение { команда } "коньць"
                 | "за" имя "отъ" выражение "до" выражение { команда } "коньць"
                 | "продолжити"
                 | "прервати"

цель            := имя | имя "по" указъ
связка          := "єсть" | "се"
составное_присваивание
                := "плюсъ єсть" | "минусъ єсть"
                 | "множити єсть" | "раздѣлити єсть" | "останъкъ єсть"
                 | "приложити" | "отъяти"
                 | "множити на" | "раздѣлити на" | "останъкъ отъ"
изменение_на_единицу
                := "плюсъ плюсъ" | "минусъ минусъ"
                 | "възрастити" | "умалити"
рядъ              := "рядъ" выражение { "и" выражение }
указъ           := целое | имя
значение_вывода := выражение | текст_без_кавычек
```

## Полный пример рядостроя

Актуальная программа находится в
[`examples/sort.slav`](../examples/sort.slav). В ней нет запятых и скобок:

```slav
възвати Сварога
възвати Ярило

Кнѧзь
    да числа єсть рядъ 3 и 1 и 2

    за ходъ отъ 0 до длъгота числа минусъ 1
        за указъ отъ 0 до длъгота числа минусъ ходъ минусъ 1
            да слѣдъ єсть указъ плюсъ 1
            аще числа по указъ паче числа по слѣдъ
                да врѣмѧнно се числа по указъ
                числа по указъ єсть числа по слѣдъ
                числа по слѣдъ се врѣмѧнно
            коньць
        коньць
    коньць

    речи съчѧти " " и числа
коньць
```

## Отладочный режим C#

Директива `#csharp` в первой строке передаёт оставшийся файл напрямую Roslyn.
На такой файл правила синтаксиса SlavLang не распространяются. Это внутренний
режим проверки backend, а не часть языка и не рекомендуемый способ написания
программ.

## Дальняя страда

Цѣль проекта — системный языкъ уровня и выше C++. Нынѣ въ ближнемъ строю суть
основания языка, а дальшая страда несетъ:

- пользовательские функции, структуры и типы;
- указатели, управление временем жизни и модель владения;
- обобщения и метапрограммирование;
- разрешение внешних призываемых модулей;
- ввод с консоли;
- динамические ряды;
- многофайловая компиляция.

Документація перечисляетъ дѣлающій синтаксисъ. Новыя силы должны входити въ языкъ явно, безъ скрытаго проброса синтаксиса C++ или C#.
